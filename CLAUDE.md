## Module Overview

`terraform-aws-iso27001` is a root Terraform module that configures AWS account-level security controls
to support ISO 27001 compliance. It is published to `registry.infrahouse.com/infrahouse/iso27001/aws`
and requires AWS provider `>= 6.0, < 7.0`.

The `.tf` files at the repo root form a single flat module (no submodules). Each file groups one
control area: `guardduty.tf`, `access-analyzer.tf`, `password_policy.tf`, `ebs_encryption.tf`, `s3.tf`,
`controltower-vpc.tf`, `contacts.tf`, `iam.tf` (the `InfraHouseLogRetention` cross-account role).

## Multi-region architecture (important)

This module intentionally does **not** use provider aliases to manage multiple regions. Instead it uses
AWS provider v6's per-resource `region` argument, driven by `var.regions` (a `list(string)`). Every
regional resource is declared with `for_each = toset(var.regions)` and sets `region = each.key`. When
adding new regional resources, follow this same pattern — do not introduce provider aliases or
per-region module instances.

Control Tower interaction: `controltower-vpc.tf` discovers Control Tower VPCs via a data source
(filtered by `tag:Name = aws-controltower-VPC`) and locks down their default security groups. The
`local.controltower_vpcs` map flattens `(region, vpc_id)` pairs so `aws_default_security_group.default`
can iterate over them without conflicting with CT-managed resources.

`locals.module_version` must be kept in sync with `.bumpversion.cfg` and the version string in
`README.md`'s Quick Start — the release Make targets handle this automatically.

## Common commands

```bash
make bootstrap      # Install Python deps from requirements.txt and install git hooks
make fmt            # terraform fmt -recursive + black tests
make lint           # black --check tests + terraform fmt -check
make validate       # terraform init -backend=false && terraform validate
make test-clean     # Run pytest against real AWS, destroy resources after (run before PRs)
make test-keep      # Run pytest against real AWS, keep resources for debugging
make release-patch  # / release-minor / release-major — git-cliff + bumpversion, must be on main
```

### Running a single test

Tests are pytest-based integration tests that create real AWS infrastructure. Configure via env vars:

```bash
TEST_PATH=tests/test_module.py TEST_FILTER="test_module and aws-6" make test-keep
```

`TEST_REGION`, `TEST_ROLE`, `TEST_SELECTOR`, `TEST_PATH`, `TEST_FILTER` are all overridable (see
`Makefile`). The test harness writes `terraform.tfvars` and `terraform.tf` into `test_data/iso27001/`
at runtime — that directory is the throwaway root module used by `pytest_infrahouse.terraform_apply`.

## Testing model

- Tests live in `tests/` and use `pytest-infrahouse` fixtures plus `infrahouse-core` for validation.
- Tests are parametrized over AWS provider versions (currently `~> 6.0`, id `aws-6`).
- They require an assumable IAM role (`--test-role-arn`) with permissions against the test account;
  `Makefile` points at `arn:aws:iam::303467602807:role/iso27001-tester` by default.
- Treat these as integration tests despite the "unit test" naming — they really create resources.

## Release flow

`make release-*` is the only supported release path. It requires `git-cliff` and `bumpversion`,
refuses to run off `main`, updates `CHANGELOG.md`, bumps `.bumpversion.cfg` (which also rewrites
`README.md` and `locals.tf`), and tags the commit. Pushing the tag triggers
`.github/workflows/release.yml` to create the GitHub Release.

## Repository-managed files (do not hand-edit)

Several files in this repo are centrally managed by the `infrahouse8/github-control` repository and
will be overwritten on next sync. Do not modify them directly — changes belong in `github-control`:

- `.claude/CODING_STANDARD.md`
- `.terraform-docs.yml`, `mkdocs.yml`, `cliff.toml`
- `hooks/` (pre-commit, commit-msg)
- `.github/workflows/` (release, docs, vuln-scanner, etc.)

The `<!-- BEGIN_TF_DOCS -->` / `<!-- END_TF_DOCS -->` block in `README.md` is auto-generated by the
terraform-docs pre-commit hook — edit the surrounding prose, not the table content.
